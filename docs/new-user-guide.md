# Registruum New User Guide

Welcome to Registruum. This guide helps new users get productive quickly with Spaces, Work Orders, team collaboration, and archive/history tools.

## Who this guide is for

- New users signing into Registruum for the first time
- Team leads onboarding members
- Anyone who needs a quick refresher on daily workflows

## Quick Start (First 10 Minutes)

1. **Sign in** to your account.
2. Open the **Dashboard** (home screen) and create your first **Space**.
3. Enter the Space and create your first **Work Order**.
4. Open **Settings > Invitations** and invite at least one teammate.
5. Use the **Notifications** bell in the top-right to review pending invite updates.
6. Use **Global Search** in the top navigation to quickly find spaces, work orders, or archive entries.

If you can complete those six steps, you are ready for normal daily use.

## Main Navigation at a Glance

- **Top navigation**
  - Space switcher (current space + quick jump)
  - Global search
  - Notifications
  - Profile menu (Settings, Billing if enabled, History, Sign out)
- **Dashboard**
  - Your list of Spaces
  - Entry point to create new Spaces
- **Inside a Space**
  - Active Work Orders
  - Team and membership areas
  - Space-specific archive and settings
- **Archive**
  - Completed/archived records and folder-based organization
- **History**
  - Your completed work history
- **Settings**
  - Profile, preferences, security/session, invitations, notifications

## Core Concepts

### Space
A Space is the top-level workspace for a team, site, or operation. Work Orders, members, and archive records belong to a Space.

### Work Order
A Work Order is an active project or task set inside a Space. It includes modules like:
- Overview
- Chats
- Members
- Documents
- Logs

### Membership and Roles
Access is role-based. Depending on your role, you may be able to:
- View only
- Collaborate on assigned items
- Manage settings and team permissions

If a page or action is missing, your role likely does not allow it yet.

## Step-by-Step: Create and Run Your First Space

1. Go to **Dashboard**.
2. Select **Create Space**.
3. Enter:
   - Space name
   - Space type
   - Optional address and photo
4. Submit the form.
5. Open the created Space from the space list or space switcher.

Tip: Use clear names (for example, building/site names) to keep search clean for your team.

## Step-by-Step: Create and Manage Work Orders

1. Enter a Space.
2. Select **Create Work Order**.
3. Fill in title, location, and initial details.
4. Open the created Work Order.
5. Use modules for ongoing collaboration:
   - **Overview** for summary and lifecycle controls
   - **Chats** for team communication
   - **Members** for assignment and visibility
   - **Documents** for files and folder organization
   - **Logs** for activity trail

### Work order lifecycle best practice

- Keep status up to date so team visibility stays accurate.
- Archive completed work instead of deleting when possible.
- Add key updates in chat/log-adjacent flows so teammates can track context.

## Inviting Teammates

You can invite users from Settings invitation controls.

Recommended process:

1. Open **Settings**.
2. Go to **Invitations**.
3. Create an invite (email or supported invite flow).
4. Assign the appropriate role.
5. Ask the user to accept the invite.

### How invite recipients find invites

- Notification bell in top nav
- Invitations section in Settings
- Invite link route if shared directly

If a user says they do not see an invite, verify they are logged into the expected account and that the invite has not expired.

## Notifications and Realtime Behavior

Registruum updates key screens automatically when relevant changes happen (for example, membership and work-order updates).

What to expect:

- Invitation count appears on the bell icon.
- Opening notifications fetches latest invitation state.
- Some pages refresh automatically after realtime events.

If updates do not appear immediately, refresh the page once and verify network connectivity.

## Using Global Search Effectively

Global search is best for quick jumps.

Tips:

- Type at least 2 characters.
- Search by space name, work order name, or archive title keywords.
- Use consistent naming conventions in Spaces and Work Orders for better search results.

## Documents and File Organization

Inside a Work Order:

- Upload files through the **Documents** module.
- Use folders to keep files grouped by purpose.
- Keep names explicit (for example, `permit-floor-2.pdf`, `inspection-2026-04-20.jpg`).

Inside Archive:

- Use folder structure and sorting/search filters.
- Keep archived records discoverable with clear titles.

## Archive and History

### Archive
Use Archive for completed records and long-term reference. You can filter by folder/space, search, and sort.

### History
Use History to review completed work items associated with your account.

## Settings You Should Configure Early

- **Profile**: name, identity details, avatar
- **Preferences**: theme and UI behavior
- **Notifications**: invite and activity preferences
- **Security / Session**: account safety and session awareness

Admins should also review invitation and permission settings regularly.

## Common New User Issues (and Fixes)

### "I cannot see a Space."
- Confirm you accepted the invitation.
- Check your account/email.
- Ask an admin to verify membership is active.

### "I can open a Space but cannot create Work Orders."
- Your role likely lacks create permissions.
- Ask a manager/admin to adjust role permissions.

### "I do not see pending invites."
- Check Notifications bell and Settings > Invitations.
- Confirm invite target account matches current login.
- Confirm invite has not expired.

### "Search returns no results."
- Use at least 2 characters.
- Try space/work-order exact naming fragments.
- Verify the item exists and you still have access.

## Team Onboarding Checklist (for Admins)

- [ ] Create Space with clear naming conventions
- [ ] Confirm default role permissions are correct
- [ ] Create 1-2 example Work Orders
- [ ] Invite core team members
- [ ] Verify each member can access required modules
- [ ] Show the team where Archive, History, and Settings are located

## Recommended Operating Habits

- Keep Work Order statuses current.
- Use chat/documents consistently in the same Work Order.
- Archive finished work instead of leaving stale active items.
- Revisit role permissions when responsibilities change.

## Need Help?

If you are blocked:

1. Capture a screenshot and route where the issue occurs.
2. Note your role and expected action.
3. Contact your internal admin or support contact with those details.

That context is usually enough to resolve access and workflow issues quickly.
